# VideoShortsAgent 架构文档

## 1. 系统概述

VideoShortsAgent 是一个 **ReAct Agent** 驱动的短视频自动剪辑系统。用户提供长视频或 URL，Agent 自主决策调用工具链，完成从下载到输出的全流程。

### 核心设计原则

- **Agent 自主决策**：LLM 驱动的 ReAct 循环，无需人工编排流程
- **Skill 即 Tool**：每个 Skill 封装为工具函数，Agent 按需调用
- **特效可配置**：Agent 根据内容风格自动选择特效组合

## 3. 模块详解

### 3.1 Agent 主循环 (`agent.py`)

ReAct（Reason + Act）模式：

1. **Reason**：LLM 分析当前状态，决定下一步行动
2. **Act**：调用工具执行
3. **Observe**：将工具结果反馈给 LLM
4. **循环**：直到 LLM 认为任务完成（不再调用工具）

最大迭代：10 轮。每轮都记录生命周期事件（`📍 [EVENT]`）。

### 3.2 工具链

| 工具 | Skill 类 | 输入 | 输出 |
|------|----------|------|------|
| `download` | `DownloadSkill` | URL | 本地视频路径 |
| `transcribe` | `TranscribeSkill` | 视频路径 | transcript.json 路径 |
| `analyze` | `AnalysisSkill` | transcript.json 路径 | clips 数组 JSON |
| `render` | `RenderSkill` | 视频路径 + analysis JSON + effects JSON | 输出视频路径 |

### 3.3 DownloadSkill (`download_skill.py`)

- 基于 **yt-dlp**，支持 1000+ 视频平台
- 使用 Chrome cookies 绕过 YouTube 反爬验证
- 输出 1080p MP4 格式

### 3.4 TranscribeSkill (`transcribe_skill.py`)

- 基于 **faster-whisper**（CTranslate2 加速版 Whisper）
- 支持 GPU 加速（CUDA）和 CPU 回退
- 输出：`[{start, end, text}, ...]` 格式 JSON

### 3.5 AnalysisSkill (`analysis_skill.py`)

- 调用 **Qwen 3.5-flash**（DashScope API）
- 提取 3-5 个精彩片段（每段 5-15 秒）
- 兼容 Qwen 思考模式（从 `reasoning_content` 回退提取 JSON）
- 使用 `str.replace()` 避免 JSON 大括号与 `str.format()` 冲突

### 3.6 RenderSkill (`render_skill.py`)

**单片段模式**：
```
FFmpeg 裁剪 → ASS 字幕生成 → 字幕烧录 → 输出
```

**多片段模式**：
```
逐段: FFmpeg 裁剪 → ASS 字幕 → 烧录
  ↓
FFmpeg xfade 转场拼接 → 最终输出
```

**转场效果**（16 种）：
`fade` / `wipeleft` / `wiperight` / `circleopen` / `circleclose` / `dissolve` / `pixelize` / `slideup` / `slidedown` 等

## 4. 特效系统

### 4.1 ASS 字幕（默认）

- FFmpeg 内置 ASS 滤镜，无额外依赖
- 支持 `\fad` 淡入淡出效果
- 字体：Microsoft YaHei 72pt，白色带黑色描边

### 4.2 Remotion 特效（可选）

位于 `remotion_effects/` 目录：

| 组件 | 效果 |
|------|------|
| `CaptionOverlay` | 字幕 spring 弹出动画 + 发光脉冲 |
| `GradientBackground` | 旋转渐变背景 + 呼吸透明度 |

通过 `effects_json` 中的 `caption_style` 和 `gradient` 控制。

## 5. 数据流

```
用户上传视频 / YouTube URL
     │
     ▼
[download] ──→ output/task_{id}/source_video.mp4
     │
     ▼
[transcribe] ──→ output/task_{id}/transcript.json
     │
     ▼
[analyze] ──→ {"clips": [{start, end, hook_text}, ...]}
     │
     ▼
[render]
  ├─ clip_0_raw.mp4 → subtitle_0.ass → segment_0.mp4
  ├─ clip_1_raw.mp4 → subtitle_1.ass → segment_1.mp4
  └─ clip_2_raw.mp4 → subtitle_2.ass → segment_2.mp4
     │
     ▼ (xfade + acrossfade)
  output/task_{id}/output_short.mp4
```

## 6. 配置

### 环境变量 (`.env`)

```bash
DASHSCOPE_API_KEY=sk-xxx    # 阿里云 DashScope API Key
```

### LLM 调试

每次 LLM 调用的输入/输出保存在 `output/task_{id}/llm_debug/`：
- `call_{n}_input.json` — 发送给 LLM 的完整消息
- `call_{n}_output.json` — LLM 返回的原始响应
